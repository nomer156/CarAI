# CodexCar

MVP современного PWA-приложения для автомобилистов. Приложение хранит цифровой профиль машины, OEM-номера деталей, цены, историю сервиса, ДТП и показывает, как может работать AI-помощник для владельца и механика.

## Что уже есть

- современный адаптивный интерфейс на `React + TypeScript + Vite`
- PWA-основа: `manifest`, `service worker`, установка на телефон/ПК
- офлайн-сохранение в `IndexedDB`
- роли `owner` и `mechanic`
- экран деталей с `OEM`, ценой и статусом замены
- раздел ТО с отметкой выполненных работ
- прозрачная сервисная история с флагом подтверждения
- MVP-чат с локальной AI-логикой
- SQL-схема для бесплатного backend на `Supabase`
- облачная интеграционная основа для `Supabase Auth`
- индекс доверия авто, документы и предложения по запчастям

## Запуск

```bash
npm install
npm run dev
```

Для включения облака создайте `.env` по примеру `.env.example`.
Для production можно взять переменные из `.env.production.example`.

Сборка production:

```bash
npm run build
```

## Рекомендуемый бесплатный стек для следующего этапа

1. `Supabase` для PostgreSQL, Auth и хранения связей между владельцем и механиком.
2. `OpenAI API` через серверный backend или edge functions, чтобы AI не работал из браузера напрямую.
3. `Storage` в Supabase для фото деталей, чеков, актов и документов.
4. PWA оставить как основной клиент, а позже упаковать в мобильное приложение через `Capacitor`.

## Этап 2

- клиент `Supabase` и вход по magic link: `src/lib/supabase.ts`, `src/lib/cloud.ts`
- расширенная схема данных и базовые `RLS`: `supabase/schema.sql`
- функция быстрого создания профиля и первой машины: `supabase/phase2_bootstrap.sql`
- миграция статуса подтверждения механика: `supabase/phase3_mechanic_approval.sql`
- миграция ролей СТО, персонала и очереди: `supabase/phase4_service_roles.sql`
- миграция администратора компании, цвета авто и brand media: `supabase/phase5_visuals_company_admin.sql`
- cloud-bootstrap для `service_admin` и `company_admin`: `supabase/phase6_staff_bootstrap.sql`
- сохранение owner/service профиля из приложения: `supabase/phase7_profile_save.sql`
- идеи и продуктовые апгрейды: `docs/PRODUCT_UPGRADES.md`
- деплой для тестеров через `Cloudflare Pages`: `docs/DEPLOY_CLOUDFLARE_PAGES.md`

## Основная идея продукта

- владелец не ищет запчасти вручную, а открывает карточку детали и сразу видит `OEM`, цену и производителя
- механик получает доступ к автомобилю клиента и фиксирует работы в истории
- при продаже авто история обслуживаний и подтвержденные работы повышают доверие
- AI-помощник берет на себя поиск, напоминания, разбор фото и чеков
